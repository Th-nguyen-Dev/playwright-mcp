# MUX-002: Wait for MCP handshake before calling `listRoots()`

## Metadata
| Field | Value |
|-------|-------|
| Status | COMPLETED |
| Priority | P0-Critical |
| Effort | S (< 1hr) |
| Assignee | Unassigned |
| Dependencies | None |
| Blocks | MUX-008 |

## Description

`connect()` in `multiplexer-server.ts` calls `this.server.connect(transport)` then immediately calls `this.server.listRoots()`. But the MCP SDK's `Server.connect()` only sets up I/O -- the handshake (`initialize` / `InitializeResult` / `initialized`) happens asynchronously on the next event loop tick. So `listRoots()` fires before the handshake completes, the request arrives at the client before `InitializeResult`, the client drops it, and the multiplexer waits 60 seconds before silently giving up.

**Impact**: Workspace root is never set, `PW_DOM_STATE_WORKSPACE` is never populated in child instance environments, and DOM state files are never written to the workspace directory.

The unused `InitializedNotificationSchema` import on line 5 is evidence of the intended fix that was never completed.

## Technical Approach

- **Architecture Layer**: Backend
- **Design Patterns**: Async handshake synchronization, Promise-based notification waiting
- **Implementation Strategy**:
  1. In `connect()`, register a notification handler for `InitializedNotificationSchema` BEFORE calling `this.server.connect(transport)`.
  2. Create a Promise that resolves when the `initialized` notification arrives.
  3. After `this.server.connect(transport)`, await the handshake Promise.
  4. Only then proceed with `this.server.listRoots()`.
  5. The existing `InitializedNotificationSchema` import (line 5) is already present and will now be used, resolving MUX-008.

## Implementation Notes

Replace the current `connect()` method with:

```typescript
async connect(transport: Transport): Promise<void> {
  // Wait for the MCP handshake to complete before making server->client requests.
  // Server.connect() sets up I/O but the handshake (initialize/InitializeResult/initialized)
  // happens asynchronously. listRoots() will fail if called before the handshake.
  const handshakeComplete = new Promise<void>(resolve => {
    this.server.setNotificationHandler(InitializedNotificationSchema, () => resolve());
  });

  await this.server.connect(transport);
  await handshakeComplete;

  // Now safe to make server->client requests
  try {
    const rootsResult = await this.server.listRoots();
    if (rootsResult.roots?.length > 0) {
      const firstRoot = rootsResult.roots[0];
      if (firstRoot.uri) {
        const workspaceRoot = this.uriToPath(firstRoot.uri);
        this.instanceManager.setWorkspaceRoot(workspaceRoot);
      }
    }
  } catch {
    // Client may not support roots - that's okay, DOM state will be disabled
  }
}
```
