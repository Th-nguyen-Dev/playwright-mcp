# MUX-007: Use `Promise.allSettled()` in `closeAll()` to prevent orphaned instances

## Description

`closeAll()` in `instance-manager.ts` (lines 220-223) uses `Promise.all()` to close all instances:

```typescript
async closeAll(): Promise<void> {
  const ids = Array.from(this.instances.keys());
  await Promise.all(ids.map(id => this.close(id)));
}
```

`Promise.all()` rejects on the first failure and abandons the remaining operations. If closing instance A throws (e.g., client already disconnected with an unexpected error), instances B, C, D are never closed.

**Consequences**:
- Orphaned Chrome child processes left running
- Orphaned Xvfb displays consuming resources
- Temporary profile directories not cleaned up
- On process exit (SIGINT/SIGTERM), the shutdown is incomplete

## Technical Approach

- **Architecture Layer**: Backend
- **Design Patterns**: Graceful degradation, best-effort cleanup
- **Implementation Strategy**:
  1. Replace `Promise.all()` with `Promise.allSettled()`
  2. After all promises settle, log any individual failures to stderr
  3. Optionally: collect all rejected reasons and throw an aggregate error after all closures complete

## Implementation Notes

```typescript
async closeAll(): Promise<void> {
  const ids = Array.from(this.instances.keys());
  const results = await Promise.allSettled(ids.map(id => this.close(id)));
  for (let i = 0; i < results.length; i++) {
    const result = results[i];
    if (result.status === 'rejected') {
      console.error(`Failed to close instance ${ids[i]}: ${result.reason}`);
    }
  }
}
```

Note: The `VirtualDisplayManager.releaseAll()` method (line 47-49 of `virtual-display.ts`) has the same pattern with `Promise.all`. Consider applying the same fix there for consistency, though it is lower risk since `release()` already has internal error handling.
