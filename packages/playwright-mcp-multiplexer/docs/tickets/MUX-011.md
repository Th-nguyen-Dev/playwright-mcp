## Description

The `exportState()` method in `auth-manager.ts` (lines 12-38) accepts `savePath` from MCP client arguments and passes it directly to `fs.promises.mkdir()` and Playwright's `storageState({ path })` without any validation:

```typescript
async exportState(instance: ManagedInstance, savePath?: string): Promise<string> {
  const outputPath = savePath ?? await this.generateOutputPath(instance.id);
  await fs.promises.mkdir(path.dirname(outputPath), { recursive: true });
  // ... writes auth state to outputPath
}
```

A malicious or misconfigured MCP client could pass `savePath: "/etc/cron.d/backdoor"` or `savePath: "../../../.ssh/authorized_keys"` and the multiplexer would write auth state (cookies, localStorage) to an arbitrary filesystem location.

**Risk assessment**: Low in practice because the MCP client is typically trusted (Claude Code), and the written content is JSON auth state (not executable). But defense-in-depth demands path validation.

## Technical Approach

- **Architecture Layer**: Backend / Security
- **Design Patterns**: Input validation, path canonicalization
- **Implementation Strategy**:
  1. Resolve `savePath` to an absolute path using `path.resolve()`
  2. Verify the resolved path starts with the configured `authDir`
  3. If validation fails, throw an error with a clear message

## Implementation Notes

```typescript
async exportState(instance: ManagedInstance, savePath?: string): Promise<string> {
  let outputPath: string;
  if (savePath) {
    // Validate savePath is within the configured auth directory
    const resolved = path.resolve(savePath);
    const authDirResolved = path.resolve(this.authDir);
    if (!resolved.startsWith(authDirResolved + path.sep) && resolved !== authDirResolved) {
      throw new Error(
        `savePath must be within the auth directory (${this.authDir}). Got: ${savePath}`
      );
    }
    outputPath = resolved;
  } else {
    outputPath = await this.generateOutputPath(instance.id);
  }

  await fs.promises.mkdir(path.dirname(outputPath), { recursive: true });
  // ... rest of method unchanged
}
```

Edge case: if `authDir` is `/home/user/.auth`, a `savePath` of `/home/user/.auth-evil/state.json` would pass a naive `startsWith` check. The `+ path.sep` ensures the path is actually inside the directory, not just a prefix match. The `=== authDirResolved` check allows writing directly to the authDir root.
